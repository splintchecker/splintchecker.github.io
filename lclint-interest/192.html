Are you using the +gnuextensions flag?

Most systems use a lot of non-standard syntax in their system header
files.  If you are using the gcc header files, the +gnuextensions flag
instructs lclint to support some of the gnu compiler extensions that may
be used in your header files.

If that doesn't work, try lclint -help parseerrors for more information
on parse errors using lclint and how to avoid them.
