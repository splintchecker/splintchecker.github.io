Re: Other Boolean types with LCLint

Currently, there is no way to do this other than using a typedef like
you suggest.  It does seems like a good idea though (and a few others
have suggested it), so I will be adding a flag to set up different
boolean names in the next minor release.

Re: Ifdef C code for LCLint checking?

Nope, there is no pre-processor variable defined by LCLint.  Note that
you can create a ~/.lclintrc file containing -D_lclint if you want one
instead of having to use the command line argument every time.
